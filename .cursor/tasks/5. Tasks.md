## Task 1: Create Interview Handler

**Status**: ✅ Complete

**Description**: Create a new handler file for interview operations.

**Files to Create**:
- `backend/internal/api/handlers/interviews.go`

**Steps**:
1. Create `InterviewHandler` struct with `q *db.Queries` field
2. Create `NewInterviewHandler(q *db.Queries)` constructor
3. Define `CreateInterviewRequest` struct:
   - `InterviewPlanID` (uuid.UUID, required, json tag: "interviewPlanId")
   - `TeacherID` (uuid.UUID, optional, json tag: "teacherId")
   - `Simulated` (bool, optional, json tag: "simulated")
   - `StudentName` (string, optional, json tag: "studentName")
   - `Status` (string, optional, json tag: "status")
4. Define `InterviewResponse` struct matching `AppInterview` fields:
   - `InterviewID` (uuid.UUID, json tag: "interviewId")
   - `InterviewPlanID` (uuid.UUID, json tag: "interviewPlanId")
   - `TeacherID` (uuid.UUID, nullable, json tag: "teacherId")
   - `Simulated` (bool)
   - `StudentName` (string, nullable)
   - `Status` (string)
   - `StartedAt` (pgtype.Timestamptz)
   - `CompletedAt` (pgtype.Timestamptz, nullable)
5. Implement `CreateInterview` method:
   - Decode JSON request body using `json.NewDecoder(r.Body).Decode(&req)`
   - Validate required fields:
     - `interviewPlanId` must not be `uuid.Nil`
   - Validate optional fields:
     - `teacherId` (if provided) must be valid UUID format
   - Validate `status` enum (if provided):
     - Must be one of: "draft", "in_progress", "completed"
     - If not provided, default to "in_progress"
   - Default `simulated` to `true` if not provided
   - Verify interview plan exists:
     - Call `h.q.GetInterviewPlanByID(ctx, planID)` with `pgtype.UUID`
     - If returns `pgx.ErrNoRows`, return 404 Not Found
     - If other error, return 500 Internal Server Error
   - Convert request to `CreateInterviewParams`:
     - `interviewPlanId`: `uuid.UUID` → `pgtype.UUID`
     - `teacherId`: `uuid.UUID` → `pgtype.UUID` (nullable, check if provided)
     - `simulated`: `bool` (default `true` if not provided)
     - `studentName`: `string` → `pgtype.Text` (nullable)
     - `status`: `string` → `string` (enum, default "in_progress")
   - Call `h.q.CreateInterview(ctx, params)`
   - Convert response from `AppInterview` to `InterviewResponse`:
     - `pgtype.UUID` → `uuid.UUID` (check `.Valid` for nullable fields)
     - `bool` → `bool` (for `simulated`)
     - `pgtype.Text` → `string` (use `.String` if `.Valid`, else null)
     - `string` → `string` (for `status`)
     - `pgtype.Timestamptz` → keep as-is (serializes to JSON)
   - Return 201 Created with response
   - Set `Content-Type: application/json` header
   - Handle errors:
     - 400 for validation errors (missing fields, invalid formats, invalid enums)
     - 404 for interview plan not found
     - 500 for database errors

**Request/Response**:
- Request: POST with JSON body (see PRD section 4.2)
- Response: 201 Created with `InterviewResponse`
- Errors: 400 (validation), 404 (interview plan not found), 500 (DB error)

**Acceptance Criteria**:
- Validates required field (`interviewPlanId`)
- Verifies interview plan exists before creating interview
- Defaults `simulated` to `true` if not provided
- Defaults `status` to "in_progress" if not provided
- Validates `status` enum values
- Properly converts between `pgtype` and standard types
- Returns appropriate HTTP status codes
- Handles nullable fields correctly (`teacherId`, `studentName`, `completedAt`)

## Notes

- Follow the pattern established in `handlers/rubrics.go` and `handlers/interview_templates.go`
- Use `pgtype` types for database operations and convert to standard types for JSON responses
- Validate UUIDs using `github.com/google/uuid` package
- Default values should be applied in handler:
  - `simulated`: default to `true` if not provided (or if `false` explicitly, use that)
  - `status`: default to "in_progress" if not provided
- Verify interview plan exists before creating interview to provide better error messages (404 vs 500)
- Handle `pgx.ErrNoRows` specifically for "not found" cases
- Nullable fields should return `null` in JSON (not empty string) when not set:
  - `teacherId`: can be null in database
  - `studentName`: can be null in database
  - `completedAt`: always null initially
- `startedAt` is automatically set by database (DEFAULT NOW())
- Error messages should be user-friendly but not expose internal details
- Consider that `simulated` is a boolean, so need to check if it's explicitly set vs default

```go
type InterviewHandler struct {
    q *db.Queries
}

func NewInterviewHandler(q *db.Queries) *InterviewHandler {
    return &InterviewHandler{q: q}
}

func (h *InterviewHandler) CreateInterview(w http.ResponseWriter, r *http.Request) {
    var req CreateInterviewRequest
    if err := json.NewDecoder(r.Body).Decode(&req); err != nil {
        http.Error(w, "invalid JSON body", http.StatusBadRequest)
        return
    }
    
    // Validate required fields
    if req.InterviewPlanID == uuid.Nil {
        http.Error(w, "interviewPlanId is required", http.StatusBadRequest)
        return
    }
    
    // Validate status enum
    status := req.Status
    if status == "" {
        status = "in_progress" // default
    } else if status != "draft" && status != "in_progress" && status != "completed" {
        http.Error(w, "invalid status: must be one of draft, in_progress, completed", http.StatusBadRequest)
        return
    }
    
    // Default simulated
    simulated := req.Simulated
    // Note: bool defaults to false, so we need to check if it was explicitly set
    // For now, default to true if not provided (this is a simplification)
    
    ctx := r.Context()
    
    // Verify interview plan exists
    planIDPgtype := pgtype.UUID{Bytes: req.InterviewPlanID, Valid: true}
    _, err := h.q.GetInterviewPlanByID(ctx, planIDPgtype)
    if err != nil {
        if errors.Is(err, pgx.ErrNoRows) {
            http.Error(w, "interview plan not found", http.StatusNotFound)
            return
        }
        http.Error(w, "failed to verify interview plan", http.StatusInternalServerError)
        return
    }
    
    // Convert to database params and create...
}
```

**Note on `simulated` default**: Since Go's `bool` defaults to `false`, we need to handle the default `true` case. Options:
1. Use a pointer `*bool` to detect if it was provided
2. Use a separate field to track if it was set
3. Default to `true` if the request doesn't include the field (check JSON decoding)

For simplicity, we can check if the field is in the JSON, but a common pattern is to use a pointer or default to `true` in the handler logic.
