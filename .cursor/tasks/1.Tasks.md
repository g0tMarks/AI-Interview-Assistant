## Task 5: Create Interview Template Handler

**Status**: âš ď¸Ź Pending

**Description**: Create a new handler file for interview template (interview plan) operations.

**Files to Create**:
- `backend/internal/api/handlers/interview_templates.go`

**Steps**:
1. Create `InterviewTemplateHandler` struct with `q *db.Queries` field
2. Create `NewInterviewTemplateHandler(q *db.Queries)` constructor
3. Define `CreateInterviewTemplateRequest` struct:
   - `RubricID` (uuid.UUID, required)
   - `Title` (string, required)
   - `Instructions` (string, optional)
   - `Config` (json.RawMessage or map[string]interface{}, optional)
   - `Status` (string, optional, defaults to "draft")
   - `CurriculumSubject` (string, optional)
   - `CurriculumLevelBand` (string, optional)
4. Define `InterviewTemplateResponse` struct matching `AppInterviewPlan` fields
5. Implement `CreateInterviewTemplate` method:
   - Decode JSON request body
   - Validate required fields (`rubricId`, `title`)
   - Validate `status` enum (if provided): "draft", "in_progress", "completed"
   - Verify rubric exists using `GetRubricByID`
   - Convert request to `CreateInterviewPlanParams`
   - Handle `config` JSON marshaling (default to `{}` if nil)
   - Call `CreateInterviewPlan` query
   - Convert response from `AppInterviewPlan` to `InterviewTemplateResponse`
   - Return 201 Created with response
   - Handle errors: 400 (validation), 404 (rubric not found), 500 (DB error)

**Request/Response**:
- Request: POST with JSON body (see PRD section 4.3)
- Response: 201 Created with `InterviewTemplateResponse`
- Errors: 400 (validation), 404 (rubric not found), 500 (DB error)

**Acceptance Criteria**:
- Validates all required fields
- Verifies rubric exists before creating plan
- Defaults `status` to "draft" if not provided
- Defaults `config` to `{}` if not provided
- Properly converts between `pgtype` and standard types
- Returns appropriate HTTP status codes

## Task 7: Create Interview Handler

**Status**: âš ď¸Ź Pending

**Description**: Create a new handler file for interview operations.

**Files to Create**:
- `backend/internal/api/handlers/interviews.go`

**Steps**:
1. Create `InterviewHandler` struct with `q *db.Queries` field
2. Create `NewInterviewHandler(q *db.Queries)` constructor
3. Define `CreateInterviewRequest` struct:
   - `InterviewPlanID` (uuid.UUID, required)
   - `TeacherID` (uuid.UUID, optional)
   - `Simulated` (bool, optional, defaults to true)
   - `StudentName` (string, optional)
   - `Status` (string, optional, defaults to "in_progress")
4. Define `InterviewResponse` struct matching `AppInterview` fields
5. Implement `CreateInterview` method:
   - Decode JSON request body
   - Validate required fields (`interviewPlanId`)
   - Validate `status` enum (if provided): "draft", "in_progress", "completed"
   - Verify interview plan exists using `GetInterviewPlanByID`
   - Convert request to `CreateInterviewParams`
   - Default `simulated` to `true` if not provided
   - Default `status` to "in_progress" if not provided
   - Call `CreateInterview` query
   - Convert response from `AppInterview` to `InterviewResponse`
   - Return 201 Created with response
   - Handle errors: 400 (validation), 404 (plan not found), 500 (DB error)
6. Implement `GetInterview` method:
   - Extract `id` from URL path parameter
   - Validate UUID format
   - Call `GetInterviewByID` query
   - Convert `AppInterview` to `InterviewResponse`
   - Return 200 OK with response
   - Handle errors: 400 (invalid UUID), 404 (not found), 500 (DB error)

**Request/Response**:
- POST Request: JSON body (see PRD section 4.4)
- POST Response: 201 Created with `InterviewResponse`
- GET Request: Path parameter `{id}` (UUID)
- GET Response: 200 OK with `InterviewResponse`
- Errors: 400 (validation/invalid UUID), 404 (not found), 500 (DB error)

**Acceptance Criteria**:
- `CreateInterview` validates all required fields
- `CreateInterview` verifies interview plan exists
- `CreateInterview` defaults `simulated` to `true`
- `CreateInterview` defaults `status` to "in_progress"
- `GetInterview` validates UUID format
- `GetInterview` returns 404 if interview not found
- Both methods properly convert between `pgtype` and standard types
- Returns appropriate HTTP status codes

## Notes

- All handlers should follow the pattern established in `handlers/rubrics.go`
- Use `pgtype` types for database operations and convert to standard types for JSON responses
- Validate UUIDs using `github.com/google/uuid` package

- Error messages should be user-friendly but not expose internal details
