## Notes

- Handler method should be added to existing `InterviewHandler` struct (no new handler needed)
- Reuse existing `InterviewResponse` struct for consistency with POST /interviews
- Follow the same error handling pattern as other GET handlers
- Use `github.com/google/uuid` package for UUID validation and parsing
- Use chi router's `URLParam` function to extract path parameters:
  ```go
  id := chi.URLParam(r, "id")
  ```
- Handle `pgtype` types carefully:
  - Check `.Valid` before accessing `.Bytes` for UUIDs
  - Check `.Valid` before accessing `.String` for Text fields
  - `pgtype.Timestamptz` can be serialized directly to JSON
- Nullable fields should return `null` in JSON (not empty string or omitted):
  - `teacherId`: can be null in database
  - `studentName`: can be null in database
  - `completedAt`: can be null (always null initially, set when completed)
- Error messages should be user-friendly but not expose internal details
- Consider trimming whitespace from path parameter before validation (though unlikely to have whitespace)

## Code Example Reference

Based on existing handler patterns and chi router usage:

```go
func (h *InterviewHandler) GetInterview(w http.ResponseWriter, r *http.Request) {
    // Extract id from path parameter
    idStr := chi.URLParam(r, "id")
    if idStr == "" {
        http.Error(w, "interview ID is required", http.StatusBadRequest)
        return
    }
    
    // Validate UUID format
    interviewID, err := uuid.Parse(idStr)
    if err != nil {
        http.Error(w, "invalid interview ID format", http.StatusBadRequest)
        return
    }
    
    ctx := r.Context()
    
    // Convert to pgtype.UUID
    interviewIDPgtype := pgtype.UUID{
        Bytes: interviewID,
        Valid: true,
    }
    
    // Query database
    interview, err := h.q.GetInterviewByID(ctx, interviewIDPgtype)
    if err != nil {
        if errors.Is(err, pgx.ErrNoRows) {
            http.Error(w, "interview not found", http.StatusNotFound)
            return
        }
        http.Error(w, "failed to retrieve interview", http.StatusInternalServerError)
        return
    }
    
    // Convert to response format
    var teacherIDResp uuid.UUID
    if interview.TeacherID.Valid {
        teacherIDResp = interview.TeacherID.Bytes
    }
    
    var studentNameResp string
    if interview.StudentName.Valid {
        studentNameResp = interview.StudentName.String
    }
    
    resp := InterviewResponse{
        InterviewID:    interview.InterviewID.Bytes, // Assuming InterviewID is always valid
        InterviewPlanID: interview.InterviewPlanID.Bytes, // Assuming always valid
        TeacherID:      teacherIDResp, // Can be null
        Simulated:      interview.Simulated,
        StudentName:    studentNameResp, // Can be null
        Status:         interview.Status,
        StartedAt:      interview.StartedAt,
        CompletedAt:    interview.CompletedAt, // Can be null
    }
    
    w.Header().Set("Content-Type", "application/json")
    w.WriteHeader(http.StatusOK)
    json.NewEncoder(w).Encode(resp)
}
```

**Note on nullable fields in JSON**: To properly return `null` in JSON for nullable fields, you may need to use pointers or custom JSON marshaling. For example:
- Use `*uuid.UUID` for `TeacherID` (nil = null in JSON)
- Use `*string` for `StudentName` (nil = null in JSON)
- `pgtype.Timestamptz` already handles null correctly when serialized

Alternatively, you can use a custom JSON marshaler or handle it in the response struct with `omitempty` and pointer types.
